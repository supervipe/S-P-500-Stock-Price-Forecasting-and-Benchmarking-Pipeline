# CSIS 4260 – Assignment 1
Stock Price Analysis, Modeling, and Dashboard

## Project Overview

This project analyzes historical daily stock prices for S&P 500 companies (2013–2018) using Python. The work is organized into three parts:

Part 1: Data Storage and Retrieval  
Compares CSV vs Parquet formats using benchmarking (file size, read time, write time) at multiple scales (1x, 10x, 100x).

Part 2: Data Manipulation, Analysis, and Modeling  
Adds technical indicators to the dataset and trains machine learning models to predict the next-day closing price. Performance is compared between Pandas and Polars.

Part 3: Dashboard  
Implements a Streamlit dashboard to visualize predictions, show model metrics, and display benchmarking results from Parts 1 and 2.

## Technical Summary

### Technical Indicators (Part 2 and Part 3)
Two indicators are computed and added as features:
- SMA (20-day Simple Moving Average): trend indicator
- RSI (14-day Relative Strength Index): momentum indicator

### Models
Two regression models are trained to predict next-day closing price:
- Linear Regression
- Random Forest Regressor

### Metrics
Regression metrics are used because the target is a continuous value (price):
- MAE (Mean Absolute Error)
- RMSE (Root Mean Squared Error)

## How to Run

### Run all parts (recommended)

From the project root directory:

python main.py

What this does:
1. Runs Part 1 benchmark and saves results into `data/processed/`
2. Runs Part 2 modeling and saves results into `data/processed/`
3. Launches Part 3 (Streamlit dashboard)

After Part 3 starts, Streamlit will print a local URL (commonly http://localhost:8501). Open it in your browser.

### Run parts individually (optional)

Part 1 only:
python part1.py

Part 2 only:
python part2.py

Part 3 only:
python -m streamlit run part3.py

## Notes

- The “next-day prediction” refers to the next trading day relative to the dataset timeline (the day after the latest available row in the data), not real-time market data.
- Ensure that `data/raw/all_stocks_5yr.csv` exists before running Part 1/Part 2/Part 3.
- If Part 3 is launched from `main.py`, it must be started using Streamlit (handled via subprocess in `main.py`). Running `part3.py` directly with `python part3.py` will not launch the dashboard correctly; use `python -m streamlit run part3.py` instead.
